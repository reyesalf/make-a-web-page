Important concepts

Lesson 1: The basics of the web and HTML

How the web works

The web is a collection of HTML documents. A browser requests to servers these documents using HTTP. A webpage is a text document written in a language called HTML. Web browsers read these documents, interpret them and display them. This video does a good job of explaining.

How coding works

Coding happens when programmer write text in a language that a computer can understand. Computers are stupid and programmers need to write exactly the way that a computer will understand by using the correct syntax.

HTML

HTML documents contains text content (which represents "what you see"), markups (which represents "what it looks like"), references to other documents and links to other pages.

HTML attributes: tags and elements

HTML documents are made of HTML elements. When we write in HTML, the browsers interpret the type of each element using HTML tags.

A tag is always contained within angled brackets. Most tags have an opening tag and a closing tag. Some tags (called "void" tags do not require a closing tag.

An element refers to everything within a set of opening and closing tags.

An attribute is a property of an HTML element. Elements can be inline or block. The difference between these elements is that block elements form an "invisible box" around the content inside of them.

Lesson 2: Creating a structured document with HTML

The tree-like structure of HTML

A tree structure is a way of representing the hierarchical nature of a structure in a graphical form.

HTML controls the structure of a web page. The browser turns HTML tags into elements in a tree structure. It knows how to do it due to a standard convention for representing and interacting with elements in HTML called Documents Object Model (DOM).

The relationship between HTML and CSS

A document written in HTML defines the structure and content of a web page. CSS controls the style of a page allowing the programmer to change how elements look on a page.

In on order to tie structure and style together, the HTML document can provide the browser with information on what style to use and where to get it.

Tools to write code

A text editor is a program that helps to type text, similar to document editors, but helping the programmer. For example, creating closing tags whenever an open tag is used. During this course it is recommended to use Scratchpad, Codepen and Sublime Text.

Lesson 3: Adding CSS style and HTML structure

CSS's style control of HTML elements

CSS controls the style of HTML elements using selectors (tags) to indicate the browser to apply rules to every element indicated by the selector. Everything indicated between the curly braces of the selector is interpreted as a rule. This rules are defined as declaration and composed by a property and the value of that property.

The style can be added to certain selectors using class selectors.

Box sizing

HTML elements are boxes and each box has 4 components: margin, border, padding and content.

It can often be hard to get the size of a box just right because there are so many components. Therefore there are two techniques that can help with sizing issues:

  • Set sizes in terms of percentages rather than pixels.
  • Set the "box sizing" attribute to "border box" for every element.

Code, test, refine

Look for natural boxes on the design and get the structure of the web page right boxifying it. Then look for repeated styles and semantic elements. Write some code, test and see how it looks against the design mockup and then make some refinements applying styles (from biggest to smallest) so that it looks closer to the design. Finally fix things and validate the code. Avoiding repetition provides consistency and it is important to prevent errors when the programmer decides to make changes.
